### Deploy = hot-copy (image is stale)
```
cd frontend && npm run build && docker cp build/. cp-api:/app/frontend/build/   # frontend
docker cp app/<f>.py cp-api:/app/app/<f>.py                                       # backend
```
**Gotchas (each bit us this session):**
1. **`kill -HUP 1` graceful reload does NOT re-run FastAPI lifespan.** It reloads workers but lifespan startup (migrations, daemons, init hooks) keeps old in-memory state. A broken lifespan import PASSES an HUP reload and only crashes on **full container restart**. ‚Üí After deleting/renaming backend modules, validate with a full restart, never just HUP.
2. **Hot-copies are ephemeral.** Image is stale vs the running container. `docker compose up -d --force-recreate cp-api` reverts to the old baked image and **wipes all hot-copies** (font, AgentFlow, prune ‚Äî everything this session). DO NOT recreate cp-api until the image is rebuilt.
3. **Durable** = `docker compose build cp-api` (bakes current source). Only then is force-recreate safe.
4. **Chat endpoint is form-encoded** (`-F "message=..."`), not JSON. `app/projects.py:843` reads `form.get("message")`.
5. Routers in `app/main.py` are **try/except guarded** ‚Üí deleting a guarded router file just unregisters it (boot-safe). But **lifespan-body imports are NOT guarded** (e.g. `init_sharepoint` at line ~292) ‚Äî those crash startup.

### Fresh-deploy hardening (AWS cold start) ‚Äî 2026-06-08
Three cold-state-only failures (warm local volumes hid them, fresh AWS volumes hit them). All fixed:
1. **`knowledge/` volume permission crash (the big one).** `knowledge/` is gitignored ‚Üí absent on a clean clone ‚Üí `/app/knowledge` never in the image ‚Üí `chown -R dash:dash /app` skips it ‚Üí Docker creates the fresh `knowledge_data` mountpoint **root-owned** ‚Üí non-root `USER dash` can't write ‚Üí `main.py:1818 makedirs('/app/knowledge/_decks')` ‚Üí `PermissionError` ‚Üí **every worker exit 3 ‚Üí restart loop**. Fix: **Dockerfile pre-creates `/app/knowledge/{_decks,embed_logos,seeds,tables,business,queries,rules}` + `/app/.gunicorn` BEFORE chown** so a fresh volume inherits `dash:dash`; `/decks` makedirs+mount made **fail-soft** in main.py; entrypoint best-effort `mkdir -p`. LANDMINE: rebuild does NOT re-chown an EXISTING root-owned volume ‚Äî drop it (`docker volume rm <proj>_knowledge_data`, it's empty) or `chown -R 999:999`.
2. **`cp-init` seeder exit 2 on cold DB.** Healthcheck was `pg_isready -U` (postmaster socket only) ‚Üí reported healthy mid-`initdb` ‚Üí first `psql` connect failed under `sh -ec` ‚Üí exit 2 (NB: psql exit 2 = connection, exit 3 = SQL error). Fix: healthcheck `-d $DB_DATABASE` + `start_period:40s`; init entrypoint has a **wait-for-postgres retry loop** (60√ó2s) before seeding.
3. **`dash_data_sources` UndefinedTable WARNING every boot** ‚Äî vestigial parent-Dash table never created here; `init_connectors` ALTERed it blindly. Fix: `to_regclass()` guard ‚Üí skip when absent (connectors feature is off in this fork anyway).

### Pharma `article_code` ‚Äî cross-table type mismatch + source corruption
The catalog‚Üîstock join key is typed **differently per table** and the stock copy is often **corrupt at source**. Two landmines:

Catalog‚Üîstock then can't link. Fix = re-upload `balance_stock` with `article_code` as **Text** in Excel (or CSV straight from source, never opened in Excel); codes must read `1000000131948`, not `1E+12`. ### Embed store-scope isolation (2026-06-08) ‚Äî 3 fixes, security-critical
A store-bound embed (`bound_scope_id`) must answer ONLY its own store. Three bugs let it leak other stores' data (cross-store, 12%‚Üí95% correct after fix):
- **Team cache MUST be per-store.** `store_id` is baked into the prompt at team-build time; the embed now passes the per-embed `synthetic_viewer` (`-embed_pk`) as `user_id` to `create_project_team` so each store gets its own cached team. `user_id=None` collides on `citypharma_None_<lang>` ‚Üí first store's prompt reused for all (60s TTL). Both chat + stream paths in `app/embed_public.py`.
- **Embeds MUST set the SQL scope.** The embed `_embed_user` needs `"via_api_key": True` or `resolve_api_scope()` returns None ‚Üí `API_STORE_SCOPE` None ‚Üí stock tool drops `WHERE site_code=‚Ä¶` ‚Üí global-total leak. With it set, the SQL filter enforces the store regardless of the prompt (defense-in-depth).
- **`store_stock_summary` uses LEFT JOIN, not INNER.** `stock_qty` is in the stock table alone; the article join only enriches `unique_articles`. INNER drops every row when `article_code` is the 1E+12 corruption (text‚âÝbigint::text) ‚Üí total 0. (low_stock stays INNER ‚Äî it `int()`s article_code.)
- Embed concurrency is NOT serialized (no single-agent lock like the gateway) ‚Äî gated by `LLM_PARALLEL_CAP_CHAT` (now 20). Test harness: `examples/embed-test/` (`setup_embeds.py` + `run_embed_test.py`). The leak was NOT caching (cache key includes site_id); reproduced with Redis flushed.

### Training pipeline + data rules (2026-06-09, latest+49)
- **Aggregate in SQL, never in-head.** `dash/instructions.py` has a HARD RULE: totals/counts/distinct MUST come from SQL `SUM`/`COUNT(DISTINCT)`/`GROUP BY`, never by summing fetched rows (the agent used to undercount many-row queries). Prefer the pre-aggregated `shop_flat` for branch-stock totals.
- **Code/ID columns land as TEXT.** `_is_id_colname` (`app/upload.py`) normalizes separators (`"Article Code"` ‚Üí `article_code`) BEFORE the id/code regex, so codes never land as int64 and cross-table joins don't hit the bigint‚Üîtext mismatch. Never raw-compare `article_code` across tables ‚Äî always `::text`, or use `shop_flat` (already normalized).
- **Derived tables are NOT trained.** `shop_flat` is excluded from the training table list (`_DERIVED_TABLES` in `app/upload.py`); it's built by `build_shop_flat` in post-hooks, not trained. `STOCK_COLS=("site_code","stock_qty","article_code")` in `dash/tools/table_sync.py` ‚Äî `article_code` is REQUIRED so the resolver can never pick `shop_flat` (which keys on `art_key`) as the stock table.
- **Training step timeouts + watchdog.** Relationship/QA verify engines run with `statement_timeout=30s` (`connect_args options`) so an un-indexable `::text` JOIN can't hang a run. `_reap_stale_runs(slug)` (`app/learning.py`) runs on every status poll: any `running`/`finalizing` run with no log progress > `STALE_RUN_MINUTES` (default 12) is auto-failed ‚Üí UI spinner can't spin forever. Run status: `running` ‚Üí `finalizing` (post-hooks) ‚Üí `done`.
- **No ML step.** AutoML removed 2026-05-23; `_task_ml()` is a no-op that PURGES any stale `ml_auto_create` row, frontend filters `ml`/`ml_auto_create`/`ml_models`. No "ML Models" badge, no machine-learning training in this product.
- **`APIGW_AUTO_PROVISION=0`** (default off, `.env` + compose). Loading multi-store stock no longer auto-mints `svc:outlet-*` API service accounts. Turn on only for per-store gateway keys.
- **Fresh-DB baseline** (`db/baseline/schema.sql` recon block): runs under `SET search_path = public, dash;` + `ADD COLUMN IF NOT EXISTS engine_model` on usage tables before the `v_usage_unified` recreate ‚Äî required or a fresh deploy fails to load. Locked `dash_projects` row still seeded by hand on a fresh DB (`ensure_locked_project()` not built).
