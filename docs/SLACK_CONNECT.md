# Slack Integration

Two independent integrations. Pick one or both.

| Integration                | Direction       | Env var(s)                              | Status      |
| -------------------------- | --------------- | --------------------------------------- | ----------- |
| Self-learning digest       | Dash → Slack    | `SLACK_LEARNING_WEBHOOK`                | shipped     |
| Slack as conversational UI | Slack ↔ Dash    | `SLACK_TOKEN` + `SLACK_SIGNING_SECRET`  | shipped     |
| Slack as a data source     | Slack → Dash KB | `SLACK_TOKEN` + connector config        | TIER 2 backlog |

## 1 — Self-learning digest (one-way notifications)

After each daily learning cycle (default 04:00 UTC), Dash posts a one-paragraph "Today we learned" message to your Slack channel.

### Webhook setup

1. https://api.slack.com/apps → **Create new app** → from scratch
2. **Incoming Webhooks** → toggle on → **Add new webhook to workspace**
3. Pick a channel (e.g. `#dash-learning`) → copy the webhook URL
4. Set env var:

```bash
SLACK_LEARNING_WEBHOOK=https://hooks.slack.com/services/T.../B.../...
```

5. Restart `dash-app`. Next cycle will post.

### Payload

```json
{"text": ":bulb: *Self-learning digest — <project_slug>*\n\n<summary paragraph>\n\n_Verified: 7 · Promoted: 2 · IQ: 312 (+8)_"}
```

Generated by `dash/learning/digest.py` after `cycle.py` consolidates verified hypotheses.

### Disable

Unset `SLACK_LEARNING_WEBHOOK` and restart. Code path is no-op when env var is unset (logs `slack notify skipped: no webhook`).

## 2 — Slack as conversational UI (full bot)

Lets users DM Dash, @mention in channels, and continue threads. Each thread = one session.

### Prereqs

- Dash on a public URL (Caddy auto-SSL or LB) OR `ngrok http 8001` for local dev
- Slack workspace install rights

### App from manifest

1. https://api.slack.com/apps → **Create new app** → **From a manifest** → JSON
2. Paste manifest below, replace `YOUR_URL_HERE` with your public base URL
3. **Create** → **Install to Workspace** → authorize

```json
{
  "display_information": {"name": "Dash", "description": "Self-learning data agent", "background_color": "#1a1a2e"},
  "features": {
    "app_home": {"home_tab_enabled": false, "messages_tab_enabled": true, "messages_tab_read_only_enabled": false},
    "bot_user": {"display_name": "Dash", "always_online": true}
  },
  "oauth_config": {
    "scopes": {
      "bot": [
        "app_mentions:read", "assistant:write",
        "channels:history", "channels:read",
        "chat:write", "chat:write.customize", "chat:write.public",
        "files:read", "files:write",
        "groups:history", "im:history", "im:read", "im:write",
        "search:read.public", "search:read.files", "search:read.users",
        "users:read", "users:read.email"
      ]
    }
  },
  "settings": {
    "event_subscriptions": {
      "request_url": "YOUR_URL_HERE/slack/events",
      "bot_events": ["app_mention", "message.channels", "message.groups", "message.im"]
    },
    "org_deploy_enabled": false,
    "socket_mode_enabled": false,
    "token_rotation_enabled": false
  }
}
```

### Credentials

- **Bot User OAuth Token** (`xoxb-...`) → `SLACK_TOKEN`
- **Basic Information → Signing Secret** → `SLACK_SIGNING_SECRET`

```env
SLACK_TOKEN=xoxb-...
SLACK_SIGNING_SECRET=...
```

Restart: `docker compose up -d --build`.

### Verify

1. App settings → **Event Subscriptions** → Request URL must show **Verified** (green). If "didn't respond", click **Retry**.
2. Slack:
   - DM Dash → reply.
   - `@Dash what's MRR?` in a channel → reply.
   - Reply in a thread → continues session, no re-mention needed.

### How it works

`app/main.py` registers Agno's Slack interface conditionally — only when both `SLACK_TOKEN` AND `SLACK_SIGNING_SECRET` are set. Each thread `ts` becomes a session id. Streaming responses post incrementally.

`SlackTools` (in `dash/team.py`) is separate — gives the Leader agent outbound tools (`send_message`, `list_channels`, `send_message_thread`, `get_channel_info`, `get_thread`, `get_user_info`, `search_messages`). Requires only `SLACK_TOKEN`.

### Updating scopes

Edit manifest → **Install App** → **Reinstall to Workspace**. New token replaces old.

## Troubleshooting

| Symptom                              | Likely cause / fix                                                |
| ------------------------------------ | ----------------------------------------------------------------- |
| Log: `slack notify skipped: no webhook` | `SLACK_LEARNING_WEBHOOK` unset — expected if disabled            |
| Log: `slack notify failed: 404`      | Webhook revoked. Regenerate, update env var, restart.             |
| Event Subscriptions "didn't respond" | Dash wasn't running on Create. Click **Retry**.                   |
| `@mention` ignored                   | Bot not invited to channel. `/invite @Dash`.                      |
| DM works, channel doesn't            | Missing `chat:write.public` OR bot not in channel                 |
| Signing failure spam in logs         | Wrong `SLACK_SIGNING_SECRET`. Copy from **Basic Information**.    |

## Bot scopes reference

| Scope                  | Why                                              |
| ---------------------- | ------------------------------------------------ |
| `app_mentions:read`    | Reply to `@Dash`                                 |
| `assistant:write`      | Slack AI assistant features                      |
| `channels:history`     | Read public channel history for context          |
| `channels:read`        | Discover public channels                         |
| `chat:write`           | Post messages                                    |
| `chat:write.customize` | Custom username / icon                           |
| `chat:write.public`    | Post without joining channel                     |
| `files:read/write`     | Read shared files; upload reports                |
| `groups:history`       | Private channel history                          |
| `im:history/read/write`| DMs                                              |
| `search:read.*`        | Search public messages, files, users             |
| `users:read[.email]`   | Identify the asker                               |
